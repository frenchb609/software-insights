# PDF Viewer - Vanilla JavaScript (Frontend Only)

A lightweight, frontend-only PDF viewer built with vanilla JavaScript. No backend server required!

## Features

✨ **Features:**

- 📁 **Upload PDFs** - Select PDF files directly from your computer
- 🔍 **View PDFs** - Display PDFs with high-quality rendering using PDF.js
- ➡️ **Navigate Pages** - Previous/Next buttons and page jump input
- 🔎 **Zoom** - Zoom in/out with reset button
- 📱 **Responsive** - Works on desktop and tablet
- 🎨 **Modern UI** - Clean design with smooth interactions
- ⚡ **No Backend** - Pure frontend solution, works everywhere!

## Setup Instructions

### Option 1: Simple File Open (Easiest)

Simply open `index.html` directly in your browser:

1. Right-click on `index.html`
2. Select "Open with" and choose your browser
3. Click "📁 Add PDFs" to select PDF files
4. That's it!

### Option 2: Local Server (Recommended)

Run a local server to access PDFs from the `pdf` folder:

#### Using Python 3:

```bash
cd c:\Users\dell\Desktop\it-information
python -m http.server 8000
```

Then open `http://localhost:8000`

#### Using Node.js:

```bash
npx http-server
```

### Option 3: GitHub Pages (Deployment)

To deploy this app to GitHub Pages with automatic PDF loading:

1. **Push to GitHub:**

   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
   git push -u origin main
   ```

2. **Enable GitHub Pages:**
   - Go to your repository Settings
   - Scroll to "Pages" section
   - Select `main` branch as source
   - Click Save

3. **The app will be live at:**

   ```
   https://YOUR_USERNAME.github.io/YOUR_REPO/
   ```

4. **Add PDFs:**
   - Add PDF files to the `pdf/` folder
   - Push to GitHub
   - GitHub Actions will automatically update `pdfs.json`
   - PDFs will appear in the app within seconds!

## How it Works

- **Local Development:** Uses File System Access API to access your local `pdf` folder
- **Local Server:** Automatically loads PDFs from `pdf/pdfs.json`
- **GitHub Pages:** GitHub Actions automatically generates `pdfs.json` whenever PDFs are added/removed
- **Upload:** Use "➕ Add PDFs" button to upload files on-the-fly (works everywhere)

## Auto-Generated PDF List

The `pdf/pdfs.json` file is **automatically generated** by a GitHub Action workflow:

- Triggered when files are pushed to the `pdf/` folder
- Lists all PDF files in alphabetical order
- Keeps the app updated without manual configuration

To trigger the workflow manually:

1. Go to GitHub Actions
2. Select "Update PDF List" workflow
3. Click "Run workflow"

## Usage

## How It Works

**Local Development (File System API):**

1. Click "📂 PDF Folder" to select your local `pdf` folder
2. The app remembers your choice using IndexedDB
3. All PDFs are automatically loaded from that folder on startup
4. Click "📁" icon next to PDFs to see they're from the folder

**Web Server/GitHub Pages:**

1. PDFs are loaded from `pdf/pdfs.json` (auto-generated by GitHub Actions)
2. Each PDF is fetched as a blob and converted to a File object
3. The app displays them just like locally selected PDFs
4. GitHub Actions automatically updates the list whenever PDFs are added/removed

**Upload:**

1. Click "➕ Add PDFs" to upload files directly
2. Files are stored in browser memory
3. Can be used alongside folder-loaded PDFs

**Rendering:**

1. Files are read as ArrayBuffers
2. PDF.js library renders pages to HTML5 Canvas
3. All processing happens client-side (browser)

## Limitations & Notes

- **Local Development**: Requires browser support for File System Access API (Chrome/Edge)
- **GitHub Pages**: PDFs must be pushed to the repository; they're served via GitHub Pages
- **Memory**: Uploaded PDFs are stored in browser memory. Refresh to clear.
- **File Size**: Very large PDFs may use significant memory
- **Persistence**: PDFs from `pdf/` folder are automatically reloaded; uploaded PDFs are not saved between sessions
- **Privacy**: All files are processed locally; nothing is sent to any server
- **Rate Limits**: GitHub API has rate limits for the automatic PDF discovery (only used if pdfs.json doesn't exist)

## Troubleshooting

### PDFs won't open

- Check browser console (F12) for error messages
- Ensure your PDF files are valid
- Try a different PDF file to confirm it's not corrupted

### Zoom not working properly

- This is normal for some PDFs with unusual dimensions
- Try zooming from 100% (Reset button)

### Button disabled after page jump

- Enter a page number between 1 and the total pages

### File upload not working

- Check browser permissions for file access
- Try using a different browser

## FAQ

**Q: Can I deploy this to GitHub Pages?**
A: Yes! Just push the repository to GitHub, enable Pages in settings, and PDFs from the `pdf/` folder will automatically appear. GitHub Actions keeps the PDF list updated.

**Q: Do I need to manually update pdfs.json?**
A: No! GitHub Actions automatically generates it whenever you push PDF files to the repository. If the workflow doesn't run, you can trigger it manually.

**Q: Can I add PDFs permanently?**
A: Yes! For GitHub Pages/web deployment, add them to the `pdf/` folder in your repository. For local use with the app open, click "📁 Add PDFs" (these are temporary unless saved to the folder).

**Q: How large can PDFs be?**
A: Depends on your browser and RAM. Typically works well with PDFs up to 100MB+. GitHub's file size limit is 100MB.

**Q: Do you collect my PDFs?**
A: No. Everything runs locally in your browser. Nothing is uploaded anywhere (unless you host on GitHub, in which case they're public).

**Q: Can I use this offline?**
A: Yes for local development - open HTML directly and select your `pdf` folder. For GitHub Pages, you need internet to fetch PDFs initially.

**Q: Does GitHub Pages work with private repositories?**
A: GitHub Pages works with private repositories, but the site will be public. PDFs will be accessible to anyone with the URL.

**Q: What if pdfs.json generation fails?**
A: The app has a fallback: it uses GitHub's API to fetch the PDF list directly. If that's also unavailable, use the "➕ Add PDFs" button to manually upload files.
